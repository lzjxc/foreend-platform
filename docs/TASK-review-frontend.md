## API 对接

```typescript
// 获取复习队列
GET /api/v1/review/queue?domain=ue5&limit=20

// 生成题目
POST /api/v1/review/card
{ atom_id: string }
→ { session_id, atom_title, question, question_type, hint }

// 提交回答
POST /api/v1/review/answer
{ session_id, answer, self_rating: 'easy'|'ok'|'hard' }
→ { is_correct, feedback, correct_answer, updated_status }

// 统计
GET /api/v1/review/stats?domain=ue5
```

## 关键交互细节

**加载题目时：**
- 点击"开始复习"后立即调用 `POST /review/card`，展示加载动画
- 加载超过 3s 显示"AI 正在思考..."文字
- 失败时显示错误提示 + 重试按钮

**答题输入框：**
- 多行文本框，最小高度 120px，支持 Shift+Enter 换行，Enter 不提交
- 字数无限制，但显示字数计数
- 空答案不允许提交（按钮 disabled）

**自评按钮：**
- 三个按钮并排，点击后立即提交（不需要额外确认）
- 提交后按钮变为选中状态，0.5s 后自动进入步骤 4

**"再练一遍错题"：**
- 取本次答错的 atoms，重新进入卡片视图，不重新生成队列

**domain 筛选 tabs：**
- 从 `GET /api/v1/review/stats` 动态获取有数据的 domain 列表
- "全部" tab 始终显示

## 样式要求

- 复习卡片视图全屏展示（隐藏侧边导航），专注模式
- 题目区域用较大字号（text-lg 或 text-xl），留足空白，避免拥挤
- 步骤切换用 Framer Motion 的 `AnimatePresence` + 滑动动画
- 反馈区域：答对用绿色背景，答错用红色背景，颜色要柔和（不要用纯色）
- 自评按钮用 emoji + 文字，大一点方便点击

---

## 验收标准

1. 复习队列正确展示，`new` 在前，`mastered` 在后
2. 点击"开始复习"后正确展示 AI 生成的题目
3. 提交回答后展示 AI 反馈，is_correct 状态视觉区分明显
4. 点击自评后正确更新进度，切换到下一题
5. 连续答对 3 次的 atom，状态在下次刷新队列时变为 mastered
6. 复习结束展示正确的总结数据
7. 步骤切换有流畅的动画
8. 加载/提交过程中有合适的 loading 状态，不会出现无响应感
